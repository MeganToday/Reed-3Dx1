## Git & GitHub Workflow

**Commit and push regularly throughout all work sessions.** After completing any meaningful unit of work (a feature, a fix, a file change), commit it with a clean, descriptive message and push to GitHub immediately. Never let significant progress sit uncommitted.

Commit message format:
- Use the imperative mood: "Add X", "Fix Y", "Update Z"
- Be specific: describe what changed and why if non-obvious
- Keep the first line under 72 characters

Example workflow:
```
git add <specific files>
git commit -m "Add player movement controls"
git push origin main
```

**Never batch unrelated changes into a single commit.** Each commit should represent one logical change.
